# libaria2 XCFramework

The libaria2 XCFramework is generated by GitHub Actions and published as a
release asset. The generated `artifacts/` and `dist/` directories are ignored
and should not be committed.

The aria2 source checkout is also ignored. Build scripts fetch upstream
`release-1.37.0` into `third_party/aria2` when needed, then apply patches from
`patches/aria2`.

## aria2 Patches

`patches/aria2/0001-apple-mobile-build-fixes.patch` carries the local changes
needed for Apple mobile builds:

- skip `SSLSetDiffieHellmanParams` on iOS-family platforms
- disable user hook execution through `fork()` on iOS-family platforms

## Build Locally

```sh
scripts/build-libaria2-xcframework.sh
scripts/verify-libaria2-xcframework.sh
scripts/package-libaria2-xcframework.sh
```

## Package Manifest Flow

`Package.swift` defaults to a remote SwiftPM binary target:

```swift
.binaryTarget(name: "libaria2", url: "...", checksum: "...")
```

For local development and CI before publishing the release asset, point the
manifest at a locally built XCFramework:

```sh
LIBARIA2_XCFRAMEWORK_PATH=artifacts/libaria2/libaria2.xcframework swift test
```

After packaging, update the manifest with:

```sh
scripts/update-libaria2-binary-target.sh \
  "https://github.com/enefry/DownloadManagerAria2/releases/download/1.0.0/libaria2.xcframework.zip" \
  "$(cat dist/libaria2.xcframework.zip.checksum)"
```

## Expected Slices

- macOS arm64, minimum macOS 13.0
- iOS arm64, minimum iOS 15.0
- iOS Simulator arm64, minimum iOS 15.0

The build is trimmed to system dynamic dependencies only. The ObjC wrapper is
linked into the same `libaria2.0.dylib`.

## Release Versioning

The `Release libaria2` workflow publishes SwiftPM source versions. It updates
`Package.swift`, commits the binary target URL/checksum, creates the semantic
version tag, and attaches `libaria2.xcframework.zip` to that same release.
Branches named like `v1.0_maint` auto-increment only the `1.0.x` line.
